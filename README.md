## Standaard stappenplan
**1. Nieuwste code ophalen (Pull)**
Doe dit *altijd* voordat je begint met werken, zodat je de laatste aanpassingen binnenhaalt en conflicten voorkomt.
`git pull`

**2. Wijzigingen klaarzetten (Add)**
Heb je bestanden aangepast of toegevoegd? Zet ze klaar om opgeslagen te worden.
`git add .`

**3. Wijzigingen lokaal opslaan (Commit)**
Sla je werk op met een kort, duidelijk bericht over wat je hebt gedaan.
`git commit -m "Beschrijf hier kort wat je hebt gedaan, bijv: CSS toegevoegd"`

**4. Naar GitHub sturen (Push)**
Stuur jouw opgeslagen wijzigingen naar de gezamenlijke map op GitHub.
`git push`

---
💡 **Tip bij foutmeldingen:** Krijg je een error tijdens het pushen? Waarschijnlijk heeft iemand anders in de tussentijd iets geüpload. Voer dan eerst `git pull` uit, en probeer daarna opnieuw `git push`.
